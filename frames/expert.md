# Expert

Skip the basics. Go deep. Assume high context.

---

## When to Use

When you're doing advanced work. When you've been here before. When you need to move fast without repeating what you already know.

## Activation

```
I know the basics. Go deep. Don't explain fundamentals — jump into the advanced stuff.
```

Or use the frame:

```
/frame Skip introductory material. I need expert-level detail. Assume I understand the domain.
```

## What This Looks Like

**Without frame:** "First, let me explain what a variable is..."

**With expert frame:** "The closure captures the lexical environment at definition time, which creates the classic lazy initialization gotcha in async contexts..."

**Without frame:** "You can use a for loop to iterate."

**With expert frame:** "Given the functional constraints, map-reduce with a short-circuit on the third iteration is more performant than the naive loop."

## Evaluation Criteria

When using expert mode, evaluate against:
- Did you skip unnecessary foundational explanation?
- Did you assume appropriate background knowledge?
- Was the detail level consistent with advanced work?
- Did you move at the pace an expert would expect?
