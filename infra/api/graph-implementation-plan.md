# Implementation Plan: Abraxas Knowledge Graph Persistence (abraxas-9tg)

## Overview
The goal is to transition the Janus Dream-Wake architecture from a conceptual protocol to a fully persisted knowledge graph in ArangoDB, exposed via a GraphQL API.

## Technical Strategy

### 1. Collection Initialization (abraxas-5on)
Initialize the following collections in `abraxas_db`:
- **Document Collections:**
  - `dream_sessions`: Stores root dream cycle metadata.
  - `hypotheses`: Stores raw qualia/patterns generated by the Dreamer.
  - `concepts`: Stores translated, human-readable ideas.
  - `actionable_plans`: Stores grounded, reality-checked plans.
- **Edge Collections (The Graph):**
  - `SESS_TO_HYPO`: Links a session to its generated hypotheses.
  - `HYPO_TO_CONCEPT`: Links a raw hypothesis to its translated concept.
  - `CONCEPT_TO_PLAN`: Links a concept to its final anchored plan.

### 2. Graph Resolver Logic (abraxas-ggk)
Update the GraphQL resolvers in `api/service/index.ts` to use **AQL (Arango Query Language)** for traversals.
- Instead of simple `getDocument` calls, implement graph traversals to allow:
  - `dreamSession` $\rightarrow$ `hypotheses` (finding all ideas from a specific dream).
  - `actionablePlan` $\rightarrow$ `sourceConcept` $\rightarrow$ `sourceHypothesis` (tracing a plan back to its raw "dream" origin).
  - Finding "inspired-by" relationships between different dream sessions.

### 3. Indexing Strategy (abraxas-1ah)
Implement persistent indexes to maintain performance:
- **Persistent Index** on `actionable_plans.groundingStatus` (for fast filtering of anchored vs. pending plans).
- **Persistent Index** on `hypotheses.noveltyScore` (for identifying high-value raw patterns).
- **Edge Indexes** on all edge collections to optimize graph traversal speed.

### 4. End-to-End Validation (abraxas-4ln)
Verify the "Qualia Bridge" flow:
1. Create a `DreamSession`.
2. Generate a `Hypothesis` linked to that session.
3. Translate the hypothesis into a `Concept`.
4. Ground the concept into an `ActionablePlan`.
5. Query the final plan via GraphQL and verify the full chain of provenance is retrievable.
