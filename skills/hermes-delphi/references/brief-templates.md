# Brief Templates

## Overview

Hermes-Delphi uses structured templates for generating client-ready intelligence briefs. All briefs follow the same basic structure but vary in depth, tone, and audience-specific content.

---

## Template Types

### Executive Brief
**Audience:** C-suite, board members, decision makers
**Length:** 1-2 pages
**Depth:** Summary-level with key points
**Tone:** Confident, decisive, action-oriented

### Technical Brief
**Audience:** Engineers, technical leads, product managers
**Length:** 3-5 pages
**Depth:** Detailed methodology, data, technical analysis
**Tone:** Precise, thorough, technically rigorous

### Strategic Brief
**Audience:** Strategists, planners, consultants
**Length:** 5-10 pages
**Depth:** Comprehensive with alternatives, scenarios, implications
**Tone:** Analytical, exploratory, balanced

---

## Brief Structure

```
1. Header
2. Summary (3 sentences max)
3. Key Findings (3-5 points)
4. Implications
5. Recommendations
6. Next Steps
7. Appendix (optional)
```

## Brief Metadata Schema

```yaml
brief_metadata:
  brief_id: uuid
  type: executive|technical|strategic
  client_id: string
  client_name: string
  created: ISO-8601
  updated: ISO-8601
  status: draft|validated|final|archived
  version: number
  
  research_sources:
    - finding_id: string
      topic: string
      confidence: sol|nox|meridian
      
  word_count: number
  reading_time_minutes: number
  
  validated: boolean
  validated_by: string (agent name)
  validated_at: ISO-8601
  
  formats_available:
    - markdown
    - html
    - pdf
```

## Quality Checklist

Before finalizing any brief:

### Content
- [ ] Summary is 3 sentences or fewer (executive) or 1 paragraph (technical/strategic)
- [ ] All key findings included with confidence labels
- [ ] Implications clearly stated
- [ ] Recommendations are actionable and specific
- [ ] Next steps are concrete and time-bound

### Sources
- [ ] All findings traceable to sources
- [ ] Source URLs included
- [ ] Confidence labels match evidence quality

### Format
- [ ] Template structure followed
- [ ] Required sections present
- [ ] Formatting consistent
- [ ] No spelling/grammar errors

### Client Context
- [ ] Client name and context correct
- [ ] Industry-specific terminology appropriate
- [ ] Recommendations relevant to client situation
